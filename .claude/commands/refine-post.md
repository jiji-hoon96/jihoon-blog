이미 작성된 블로그 글을 jihoon 스타일로 리파인해주세요.

## 작업 순서

1. 대상 파일을 읽습니다.
2. 현재 글의 **구조, 톤, 표현**을 분석합니다.
3. 글에 포함된 기술적 내용의 **사실 관계와 최신성을 검증**하기 위해 웹 검색으로 리서치합니다.
4. 아래 체크리스트에 따라 개선점을 파악하고, 리서치 결과를 반영하여 수정합니다.
5. 수정된 글을 원본 파일에 반영합니다.

## 리파인 체크리스트

### 구조
- [ ] 도입부에 "이번 포스팅에서는 [주제]에 대한 이야기를 해보려고 한다." 패턴이 있는가?
- [ ] 도입부에서 개인적 경험이나 동기가 2-3문단에 걸쳐 설명되는가?
- [ ] 소제목이 질문형과 선언형의 적절한 혼합인가?
- [ ] 섹션 전환이 질문을 통해 자연스럽게 이루어지는가?
- [ ] 결론부가 논지를 압축 요약하고 독자에게 말을 건네는가?

### 톤과 어조
- [ ] 한다체(해체)가 일관되게 사용되는가?
- [ ] 1인칭이 "필자"로 통일되어 있는가?
- [ ] 괄호 안 유머/코멘트가 적절히 배치되어 있는가? (과하지도, 부족하지도 않게)
- [ ] 양보-주장 구조("물론 ~이지만")가 활용되는가?

### 기술 설명
- [ ] 전문 용어에 풀이가 병기되어 있는가?
- [ ] 개발자가 직관적으로 이해할 수 있는 비유가 사용되었는가?
- [ ] 비유의 한계가 적절히 언급되는가?
- [ ] 주장에 레퍼런스(논문, 문서 등)가 뒷받침되는가?

### 사실 관계 및 최신성
- [ ] 기술적 주장(성능 수치, 동작 원리, 비교 등)이 정확한가?
- [ ] 버전, 생태계 현황 등이 현재 시점 기준으로 최신인가?
- [ ] 리서치를 통해 보강할 수 있는 내용(최신 동향, 대안, 사례)이 반영되었는가?
- [ ] 기술 용어의 정의와 약어 풀이가 정확한가?

### SEO
- [ ] `title`이 짧고 명확한가? (사이트 H1/카드에 표시되므로 깔끔하게)
- [ ] 검색 노출을 강화하고 싶은 글이라면 `seoTitle` (50~60자, 핵심 키워드 포함)을 추가했는가?
  - `seoTitle`은 `<title>` 태그, OG, 트위터, JSON-LD `alternativeHeadline`에 반영됨
  - 짧은 `title`을 유지하면서 검색엔진엔 풍부한 키워드를 노출할 수 있는 장치
  - 예: title `"추상화"` + seoTitle `"프론트엔드 추상화 — 좋은 코드를 위한 설계 원칙"`
- [ ] 프론트매터에 `description` (120~160자, 핵심 키워드 포함)이 있는가?
- [ ] 프론트매터에 `keywords` (구체적 롱테일 키워드 5~8개)가 있는가?
- [ ] description이 글의 핵심 가치를 명확하게 전달하는가?

### 문장과 문단
- [ ] 긴 설명문 후 짧은 단문으로 리듬감이 살아 있는가?
- [ ] 1문단이 1-3문장 내로 구성되어 있는가?
- [ ] "~것이다", "~인 것이다" 문미가 적절히 활용되는가?
- [ ] 코드 예시, 인용문 등 시각적 요소가 적절히 배치되어 있는가?
