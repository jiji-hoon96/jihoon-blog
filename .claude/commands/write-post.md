사용자가 작성한 초안을 jihoon 스타일의 기술 블로그 글로 정리해주세요.

## 작업 순서

1. 사용자의 초안을 읽고 **핵심 주제와 논지**를 파악합니다.
2. 주제와 관련된 **최신 기술 동향, 레퍼런스, 사례**를 웹 검색으로 리서치합니다.
3. 초안 내용과 리서치 결과를 결합하여, 아래 "문체 가이드"에 따라 글의 구조, 톤, 표현을 다듬습니다.
4. 결과물을 `content/` 디렉토리의 적절한 날짜 폴더에 마크다운으로 작성합니다.

## 리서치 가이드

- 초안에 포함된 기술적 주장(성능 수치, 비교, 동작 원리 등)의 **사실 관계를 검증**합니다.
- 주제와 관련된 **최신 버전, 변경사항, 생태계 현황**을 조사하여 글의 시의성을 확보합니다.
- 업계 전문가의 발언, 공식 문서, 벤치마크 등 **레퍼런스 기반 서술**에 활용할 자료를 수집합니다.
- 리서치 결과 중 초안에 없던 내용이라도, 주제의 깊이를 더하거나 독자에게 유용한 맥락이라면 적극 반영합니다.
- 기술 용어의 정확한 정의와 약어 풀이를 확인합니다.

## 문체 가이드 (jihoon 스타일)

### 톤과 어조
- **한다체(해체)** 사용: "~한다", "~이다", "~것이다"
- 1인칭은 **"필자"**를 일관되게 사용 ("나" 대신)
- 기본 톤은 **차분하고 논리적**이되, 괄호 안 필자의 개인적인 생각과 의문을 자연스럽게 표현
  - 예: `(쓰고나니 뭔가 소개팅 같다)`, `(속된 말로 발작버튼이 눌릴 수도 있다)`
- **양보-주장 구조**를 활용: "물론 ~이지만, 필자 생각은 다르다"

### 도입부 패턴
1. **선언 문장**: "이번 포스팅에서는 [주제]에 대한 이야기를 해보려고 한다."
2. **개인적 맥락**: 2-3문단에 걸쳐 왜 이 주제를 다루게 되었는지 설명
3. **독자 공감**: 본인이 처음 이 개념을 접했을 때의 경험담으로 시작

### 소제목
- **질문형**과 **선언형**을 혼용: "TCP가 왜 느리다고 하는 걸까?", "작성자에서 의사결정권자로"
- h2, h3 두 단계까지만 사용
- 소제목은 독자의 궁금증을 자극하는 방향으로

### 주제 전환
- 직전 섹션의 결론을 **질문으로 변환**해서 다음 섹션으로 연결
  - 예: "그렇다면 지금 우리는 어떤 방향으로 미래를 준비해야 할까?" -> 다음 섹션
- "여기까지 읽으면 자연스럽게 떠오르는 질문이 하나 있다."

### 기술 개념 설명
- **영어 약어/전문 용어는 반드시 풀어서 설명**: "DRI(Directly Responsible Individual)는..."
- **비유와 메타포를 적극 활용**: 개발자가 직관적으로 이해할 수 있는 비유 사용
  - 예: "작업 기억은 일종의 고정 크기 스택이라고 볼 수 있다"
- 비유 후에는 **한계를 짚어주기**: "다만 실제로는 ~이기 때문에..."
- 학술 논문, RFC 등 **레퍼런스 기반 서술** 선호

### 문장 구조
- **긴 설명문 + 짧은 단문의 리듬감**: 2-3문장 설명 후 "이런 상황인 것이다."로 마무리
- **질문형 문장**을 전환 장치로 적극 사용
- **"~것이다", "~인 것이다"** 문미를 활용해 단정적이면서도 설명적인 뉘앙스
- 1문단 = 1-3문장, 문단 사이 빈 줄로 여유롭게

### 결론부 패턴
- 글 전체 논지를 1-2문단으로 **압축 요약**
- 독자에게 **말을 건네는 형식**으로 마무리
- 미래에 대한 **겸손한 불확실성** 표현
  - 예: "정답은 없지만... 이 글을 읽는 독자 분들도 각자만의 답을 찾아보기를 바란다."

### 유머 장치
- **괄호 안 코멘트**: 본문의 진지한 톤 vs 괄호 안의 가벼운 톤 대비
- **가상 대사 인용**: "AI가 작성한거라 전 왜 이렇게 됐는지 모르겠는디요"
- **인터넷 밈/속어**: "꼴랑", "알잘딱" 등을 과하지 않게 톤 전환이 필요한 지점에서만

### 블로그 포스트 프론트매터 형식
```yaml
---
emoji: [적절한 이모지]
title: '[포스트 제목 — 짧고 명확하게]'
seoTitle: '[선택: 검색엔진용 긴 제목, 50~60자, 핵심 키워드 포함]'
date: '[YYYY-MM-DD]'
categories: [공백으로 구분된 카테고리]
description: '[검색 결과 스니펫용 메타 디스크립션, 120~160자, 핵심 키워드 포함]'
keywords: '[쉼표로 구분된 검색 타겟 키워드 5~8개, 롱테일 키워드 위주]'
---
```

### SEO 작성 가이드
- **title**: 사이트에 표시되는 H1/카드 제목. 짧고 본질적으로. 예: `"추상화"`, `"도메인 모델"`
- **seoTitle** (선택): 짧은 `title`로 사이트 가독성을 지키면서 검색엔진엔 풍부한 키워드를 노출하고 싶을 때 사용.
  - `<title>` 태그, OG, 트위터, JSON-LD `alternativeHeadline`에 모두 반영됨
  - 50~60자 권장 (구글 검색결과에서 잘리지 않는 길이)
  - 메인 키워드 + 보조 키워드 + 액션/혜택 패턴으로 작성. 예: `"프론트엔드 추상화 — 좋은 코드를 위한 설계 원칙"`
  - 미지정 시 `title`이 자동 대체되므로, 키워드 강화가 필요한 글에만 추가하면 됨
- **description**: 글의 핵심 내용과 독자가 얻을 가치를 명확히 전달. 주요 검색 키워드를 자연스럽게 포함.
- **keywords**: 구체적이고 검색량이 있는 롱테일 키워드 위주. 한글/영문 혼용 가능.
